Developing Accessible Web Pages

Developing accessible web pages is very important as a web developer. As our world is becoming dependent on the Internet for more of our daily tasks, we have to take into account accessibility features and make sure everyone can view the content we are serving out. I think that all web developers should be aware of accessiblity features and apply those concepts when writing code.

According to this site, 2,250 lawsuits were filed for inadequate web accessiblity in 2018. Suits range from large companies such as Netflix to small companies. I think that people should incorporate accessbility features because it's the right thing do, not because they fear a lawsuit (although if that's what it takes so be it.) With the help of this blog post, I am going to dicuss how to apply a few important accessibility features.

Tab Navigation

One important accessibility feature is tab navigation, or the ability to navigate the page using the physical tab button on your keyboard. This feature is important for people who may not be able to view the screen, but have the ability to use the keyboard. So, let's try it out. I'm going to add two different buttons below.

You can loop through the buttons on the screen using the tab button on the keyboard and go to the actual page by hitting the enter button on the keyboard. As important as it is to have the ability to tab between buttons, I think the feature of navigating between pages using those buttons is just as important. If you tab through this screen, you can see that you can tab through the buttons, but not hyperlinks. Because of this, buttons are of better use to navigate between screens instead of plain hyperlinks.

Color Constrast

People view color in different ways, therefore everyone on that spectrum should have the ability to view the content on the webpage regardless of their abiity to see color in the "same way" as everyone else. Header Navigation

The next important feature of an accessible website is having clearly labeled headers. I had to fix my code a bit to comply with these standards. It doesn't sound like a pressing issue when coding, but it is really important for people who rely on readers to go through web pages. For instance, you should not include two different h1 headers on the page as that can get confusing for people who rely on readers.

On this web page, the big header at the top uses an h1 tag to relay it's importance as the main header. My name beneath it as well as the topic of this blog both use h2 tags because they're important, but not as important as h1. The subsequent "sub-headers" on the page all use the h3 tag because they are the next "level" in header importance. If I wanted to break up this section into two subsequent sections, I would label both of those headers using the h4 tag because that would be the next level of headers.

Takeaways

I enjoyed learning about the topic of website accessibility, and I will definitely be applying it beyond this class. Developing accessible websites is necessary and will contribute to a more inclusive online world where everyone feels comfortable accessing content.
